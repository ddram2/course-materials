# Session 3. 학생 발표자 가이드 — AI 활용 Tip 나누기

> 발표자에게 사전 배포 | 워크숍 당일 세션 3 (15:35–16:05)

---

## 발표 개요

- **시간**: 1인당 7분 (발표 5분 + 질의 2분)
- **목적**: 각자의 연구 맥락에서 AI를 어떻게 활용하고 있는지 구체적 경험을 공유
- **청중**: 환경보건학과 석·박사 대학원생 + 학부생
- **형식**: 자유 (슬라이드, 화면 시연, 구두 발표 모두 가능)

---

## 발표 구성 가이드

아래 4가지 항목을 중심으로 준비해 주세요. 형식은 자유입니다.

### 1. 내 연구에서 AI를 쓰는 장면 (1분)
> "나는 어떤 상황에서 AI를 쓰는가?"

- 예) 논문 검색·요약, 코드 작성·디버깅, 영어 교정, 데이터 전처리, 문헌 정리 등
- 구체적인 도구명을 밝혀주세요 (ChatGPT, Claude, Copilot, Elicit, Zotero+AI 등)

### 2. 가장 유용했던 경험 (2분)
> "AI 덕분에 확실히 효율이 올라간 사례"

- 어떤 작업을, 어떤 AI 도구로, 어떻게 해결했는지
- Before/After가 있으면 좋습니다 (예: "전에는 2시간 걸리던 것이 20분으로")
- **가능하면 화면을 보여주세요** — 실제 프롬프트, 실제 결과

### 3. 실패했거나 주의해야 할 경험 (1분)
> "AI가 틀렸거나, 기대와 달랐던 사례"

- 예) 존재하지 않는 논문 인용(hallucination), 통계 해석 오류, 코드 버그 등
- 그 경험에서 배운 교훈은 무엇인가

### 4. 동료에게 추천하는 Tip 하나 (1분)
> "하나만 추천한다면?"

- 특정 도구, 특정 프롬프트 패턴, 특정 워크플로우 등
- 가능한 한 구체적으로 (도구명 + 사용법 + 효과)

---

## 발표 예시

> **"저는 역학 전공으로, 주로 R로 통계 분석을 합니다.**  
> 최근 Claude에게 R 코드 작성을 맡기고 있는데, 가장 유용했던 건 복잡한 Table 1을 자동 생성하는 코드를 만들어 준 것입니다. (화면 시연)  
> 다만, mixed model에서 random effect 구조를 잘못 지정한 적이 있어서 항상 모델 구조를 직접 확인합니다.  
> 추천 Tip: 코드를 요청할 때 '분석 목적 + 데이터 구조 + 원하는 출력 형식'을 함께 알려주면 정확도가 크게 올라갑니다."

## 준비 시 유의사항

- **5분을 넘기지 마세요** — 핵심만 전달하는 것이 좋습니다
- **완벽할 필요 없습니다** — 실패 경험도 훌륭한 공유입니다
- **시연 > 슬라이드** — 실제 화면을 보여주는 것이 효과적입니다
- **발표 후 질의에 대비하세요** — "그 도구 유료인가요?", "어떤 모델을 쓰나요?" 같은 질문이 올 수 있습니다
