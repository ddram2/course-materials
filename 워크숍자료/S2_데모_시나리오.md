## IDE 소개 (5분) — 강의

### 스크립트 요점

> "IDE는 원래 개발자의 도구였습니다. 하지만 Vibe Coding 시대가 되면서, 코드를 한 줄도 모르는 연구자도 IDE에서 자연어로 데이터 분석, 문서 편집, 시각화를 할 수 있게 되었습니다."

**전달할 내용**:

1. **Vibe Coding이란**: 코드를 직접 쓰지 않고, 자연어로 AI에게 지시하여 코딩하는 방식 (Andrej Karpathy, 2025)
2. **왜 IDE인가**: ChatGPT 채팅창과 달리, IDE에서는 **파일을 직접 읽고 수정**하고, **코드를 즉시 실행**하고, **여러 문서를 맥락으로 활용**할 수 있다
3. **추천 IDE 3종**: Antigravity(무료, 오늘 실습), Windsurf(시연), Cursor — 모두 VS Code 기반
4. **오늘의 구성**: 진행자가 Windsurf에서 고급 기능을 시연하고, 학생 여러분은 Antigravity에서 직접 실습한다

> 상세: [[S2_왜_IDE인가]]

## 시연① — .md 특정 문장/문단 지정 수정 (8분) — Windsurf

### 시나리오
[[S2_데모_영어초고.md]]를 Windsurf에서 열고, AI(Cascade)에게 **특정 부분만** 수정을 요청한다.

### 시연 흐름

**Step 1 — 특정 문단 선택 후 수정 요청 (inline edit):**
> 첫 번째 문단을 드래그로 선택한 뒤:
```
이 문단을 Environmental Health Perspectives 저널 스타일에 맞게 더 학술적인 톤으로 다듬어 줘.
```
→ AI가 선택한 부분만 수정 제안 → Accept/Reject 시연

**Step 2 — 문장 하나만 지정 수정:**
> "OR = 1.45" 문장을 선택한 뒤:
```
이 통계 표현을 APA 스타일 (7th edition)로 바꿔줘.
```

**Step 3 — 전체 파일 문법 검토:**
```
이 영어 초고에서 문법 오류와 부자연스러운 표현을 찾아,
원문과 수정본을 나란히 마크다운 표로 보여줘.
```

### 학생에게 전할 메시지
- **ChatGPT와의 차이**: 채팅창에 복사-붙여넣기 하는 것이 아니라, **파일 안에서 원하는 부분만** 지정하여 수정할 수 있다
- AI가 문법뿐 아니라 학술적 톤/스타일도 조정 가능하다
- **경고**: AI에게 "final-touch만 맡기는 것"은 자기 글쓰기 역량 축적을 방해한다 (→ 세션 4 연결)

## 시연②→실습 — R Vibe Coding: mtcars (12분) — Windsurf → Antigravity

### R Extension 설치 실습 (현장, 2분)

> R이 컴퓨터에 설치되어 있다는 전제하에, Antigravity에서 R 코드를 실행할 수 있도록 Extension을 설치한다.

**학생에게 안내할 단계:**

1. Antigravity 좌측 사이드바 → **Extensions** 아이콘 클릭 (또는 `Cmd/Ctrl + Shift + X`)
2. 검색창에 **"R"** 입력
3. **"R" by REditorSupport** 를 찾아 **Install** 클릭
4. (선택) **"R Debugger"** 도 설치하면 디버깅 가능
5. 설치 후 Antigravity 재시작 (또는 Reload Window)

> **확인**: 터미널에서 `R --version` 입력 → 버전이 출력되면 R 연동 준비 완료
> R이 설치되지 않은 학생은 코드 생성만 보고, 나중에 RStudio에서 실행해도 된다.

### 진행자 시연 (7분, Windsurf)

**Step 1 — Summary Table 생성:**
```
R의 내장 데이터 mtcars를 사용해서 summary table을 만들어줘.
gt 패키지를 사용하고, 각 변수의 평균, 표준편차, 최소, 최대를 포함해줘.
```
→ AI가 R 스크립트 생성 → 실행 → 표 출력 시연

**Step 2 — ggplot 탐색적 시각화:**
```
mtcars에서 mpg와 wt의 관계를 ggplot 산점도로 그려줘.
cyl(실린더 수)별로 색상을 구분하고, 회귀선도 추가해줘.
```
→ 산점도 생성 시연

**Step 3 — 회귀분석:**
```
mtcars에서 mpg를 종속변수로, wt, hp, cyl을 독립변수로 하는
다중 회귀분석을 수행하고, 결과를 깔끔한 표로 정리해줘.
broom 패키지의 tidy() 형식으로.
```
→ `lm()` + `broom::tidy()` 결과 시연

### 학생 실습 (3분, Antigravity)

> "이제 여러분 차례입니다. Antigravity 채팅창에 다음 프롬프트를 직접 입력해 보세요."

**실습 프롬프트 (화면에 표시):**
```
mtcars 데이터에서 자동(am=0)과 수동(am=1) 변속기의 
mpg 차이를 boxplot으로 그려줘. ggplot 사용.
t-test 결과도 함께 보여줘.
```

> 코드가 생성되면 실행해 보세요. 에러가 나면 AI에게 "에러가 났어, 수정해줘"라고 해 보세요.

### 학생에게 전할 메시지
- **코드를 한 줄도 직접 쓰지 않았다** — 이것이 Vibe Coding이다
- AI가 생성한 코드가 맞는지 **결과를 반드시 검증**해야 한다
- R ↔ Python 전환도 "이걸 Python으로 바꿔줘" 한 줄이면 된다

## 시연③ — 여러 문서 분석 → 쟁점 정리 → 최종 문서 (10분) — Windsurf

### 시나리오
진행자가 자신의 폴더에 있는 여러 문서를 AI에게 읽히고, 분석·정리·사실확인을 거쳐 최종 문서를 만드는 과정을 시연한다.

### 시연 흐름

**Step 1 — 여러 파일 지정 후 요약:**
```
다음 3개 파일을 읽고 각각 핵심 내용을 3줄로 요약해줘:
- S4_기조발제_노트.md
- S4_패널토론_진행가이드.md
- S4_AI활용_자기점검_체크리스트.md
```

**Step 2 — 쟁점 추출:**
```
이 3개 문서에서 공통적으로 다루는 쟁점(이슈)을 3가지로 정리해줘.
각 쟁점에 대해 근거가 되는 내용을 인용해줘.
```

**Step 3 — 사실확인 (웹 검색):**
```
위 쟁점 중 "AI 시대의 Cognitive Debt(인지부채)"에 대해
최근 학술 문헌에서 어떻게 논의되고 있는지 검색해줘.
```
→ MCP(brave-search 등)로 웹 검색 → 결과를 맥락에 추가

**Step 4 — 최종 문서 작성:**
```
위 분석과 검색 결과를 종합하여,
"AI 시대 대학원생의 연구 역량"이라는 제목으로
800자 내외의 에세이 초안을 작성해줘.
마크다운 형식으로, 소제목 포함.
```

### 학생에게 전할 메시지
- IDE는 **여러 문서를 한꺼번에 맥락으로** 다룰 수 있다 — ChatGPT에 일일이 붙여넣지 않아도 된다
- 웹 검색으로 **사실관계를 확인**한 후 문서에 반영하는 워크플로우가 가능하다
- 이런 고급 기능은 Windsurf + MCP/SKILL에서 가능하며, 향후 다른 IDE에서도 확대될 것이다

## (시간 허용시) 시연④ — MCP/SKILL 문헌 검색 (5분) — Windsurf

### 시나리오
Windsurf의 Cascade에서 MCP(PubMed) 또는 SKILL(literature-review)을 호출하여, IDE 안에서 논문을 검색하고 결과를 .md 파일로 정리하는 과정을 시연한다.

### 시연 프롬프트

```
PubMed에서 "PFAS thyroid function" 관련 최근 5년간(2021–2026)의
주요 논문을 5편 검색하고, 각 논문의 제목, 저자, 연도, 주요 결과를
마크다운 표로 정리해줘.
```

### 학생에게 전할 메시지
- IDE가 **코드 편집기를 넘어 연구 플랫폼**이 될 수 있다
- MCP(Model Context Protocol)·SKILL은 AI의 기능을 확장하는 플러그인 같은 것이다
- 현재 Windsurf(Claude)에서 가장 잘 지원되며, 다른 IDE에서도 점차 확대될 전망이다

## 실습 — PubMed 워드클라우드 만들기 (5분) — Antigravity

### 학생 실습 프롬프트

> "Antigravity 채팅창에 다음을 입력해 보세요."

```
PubMed에서 'PFAS biomonitoring'으로 논문을 검색하고,
검색된 논문들의 키워드를 D3.js 워드클라우드로 시각화하는
HTML 웹페이지를 만들어 줘.
NCBI E-utilities API를 사용하고,
워드를 클릭하면 관련 논문 목록이 나오게 해줘.
```

> AI가 index.html, app.js, style.css 등의 파일을 생성한다.
> 브라우저에서 index.html을 열면 워드클라우드가 표시된다.
> 검색어를 바꿔서 자기 연구 분야의 키워드 맵을 만들어 볼 수도 있다.

**완성 예시**: `pubmed_wordcloud/index.html` (진행자 시연 겸용)

### 학생에게 전할 메시지
- 자연어 한 줄로 **API 연동 웹앱**까지 만들 수 있다 — Vibe Coding의 가능성
- 검색어만 바꾸면 자기 분야의 키워드 트렌드를 시각적으로 파악할 수 있다
- 패키지·라이브러리가 없으면 AI가 설치나 CDN 링크를 안내해 준다

## 프롬프트 비교 + Q&A (5분)

### 나쁜 프롬프트 vs 좋은 프롬프트

| 구분 | 나쁜 프롬프트 ❌ | 좋은 프롬프트 ✅ |
|---|---|---|
| 모호함 | "이 데이터 분석해 줘" | "mtcars에서 mpg와 wt의 관계를 ggplot 산점도로 그려줘, cyl별 색상 구분" |
| 맥락 부재 | "논문 써줘" | "이 3개 문서의 공통 쟁점을 정리하고, 800자 에세이 초안을 작성해줘" |
| 역할 미지정 | "이거 고쳐줘" | "너는 환경보건학 분야 영어 논문 에디터야. 이 문단의 학술적 표현을 개선해 줘" |
| 출력 형식 미지정 | "결과 보여줘" | "broom::tidy() 형식으로 회귀분석 결과를 마크다운 표로 정리해줘" |

> **핵심**: 좋은 프롬프트 = **맥락 + 구체적 지시 + 출력 형식**
