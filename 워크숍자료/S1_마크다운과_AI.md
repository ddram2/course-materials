# 마크다운과 AI — 왜 .md가 LLM 시대의 기본 문서 형식인가

> 세션 1 보충자료 | 2026.02.23 워크숍

---

## 핵심 메시지

> **마크다운을 아는 것은 AI를 잘 쓰기 위한 기초 체력이다.**

---

## 1. LLM의 입출력 언어 = 마크다운

ChatGPT, Claude, Copilot 등 주요 Large Language Model(대규모 언어모델)은 **마크다운 형식으로 응답**한다.

| AI 도구 | 응답 형식 |
|---|---|
| ChatGPT (OpenAI) | 마크다운 (제목, 목록, 표, 코드블록) |
| Claude (Anthropic) | 마크다운 |
| Copilot (GitHub/Microsoft) | 마크다운 |
| Gemini (Google) | 마크다운 |

LLM의 학습 데이터(training data)에 마크다운 문서가 대량 포함되어 있기 때문이다. 따라서 마크다운 문법을 알면 **AI의 출력을 읽고 편집하는 속도가 빨라진다**.

---

## 2. .md 파일은 AI에 바로 입력 가능하다

| 파일 형식 | AI에 입력하려면? |
|---|---|
| **.md (마크다운)** | 그대로 복사·붙여넣기, 또는 파일 첨부 |
| .docx (Word) | 변환 필요 (텍스트 추출 → 서식 손실 가능) |
| .hwp (한글) | 변환 매우 어려움 (AI가 직접 읽지 못함) |
| .pdf | 텍스트 추출 필요 (이미지 기반 PDF는 OCR 필요) |

**Plain text(일반 텍스트)** 기반인 마크다운은 어떤 AI 도구에서든 바로 처리할 수 있다.

---

## 3. 구조화된 프롬프트 → 더 나은 AI 응답

마크다운의 **헤딩(#)**, **목록(-)**, **코드블록(```)** 등으로 프롬프트를 구조화하면 LLM이 맥락을 더 잘 파악한다.

### 나쁜 프롬프트 (구조 없음)

```
이 논문 요약해줘 Kim et al 2024 PFAS 노출과 갑상선 기능 이상 연구인데 
방법론 비판도 해줘 그리고 우리 연구에 시사점도
```

### 좋은 프롬프트 (마크다운 구조)

```markdown
# 요청
아래 논문 노트를 바탕으로 세 가지를 수행해 주세요.

## 논문 정보
- **저자**: Kim et al. (2024)
- **주제**: PFAS 노출과 갑상선 기능 이상

## 수행할 작업
1. **요약**: 연구 목적, 방법, 주요 결과를 3문장으로 요약
2. **방법론 비판**: 연구 설계의 한계를 2가지 지적
3. **시사점**: 우리 연구(종단 코호트)에 반영할 점 제안
```

**차이**: 구조화된 프롬프트는 AI가 작업 범위를 명확히 인식하고, 출력도 구조적으로 반환한다.

---

## 4. 마크다운 습관 = AI 활용 역량

| 습관 | AI 활용에서의 효과 |
|---|---|
| 논문 노트를 .md로 정리 | AI에게 맥락(context) 제공 → 정확한 요약·비판 |
| 연구 계획을 .md로 작성 | AI와 반복 대화하며 구체화 가능 |
| 실험 기록을 .md로 남김 | AI가 기록 기반으로 분석 코드 생성 |

> 마크다운으로 자료를 축적할수록, AI가 **내 연구 맥락**을 파악하는 데 유리해진다.  
> 이것이 Obsidian의 [[S1_Obsidian_플러그인_안내#Smart Composer|Smart Composer]]와 연결되는 지점이다.
